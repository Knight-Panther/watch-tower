- The pipeline stages described omit existing active stages `embedding` and `scoring_failed`, and the system already uses `posting` / `posting_failed`. `packages/worker/src/processors/semantic-dedup.ts:55-118`, `packages/worker/src/processors/llm-brain.ts:411-418`, `packages/worker/src/processors/distribution.ts:124-283`

**Risks / potential issues**
- If translation is inserted “after scoring” without disabling immediate distribution, approved articles can be posted before translation completes (or even when Georgian mode is enabled). `packages/worker/src/processors/llm-brain.ts:379-401`, `packages/worker/src/processors/distribution.ts:226-234`
- Georgian posting mode would still post English today because distribution/maintenance formatters always use `title` + `llm_summary`; without enforced blocking, posts go out in the wrong language. `packages/worker/src/processors/distribution.ts:226-234`, `packages/worker/src/processors/maintenance.ts:452-459`
- `app_config` writes currently coerce numbers to strings and booleans to booleans; the proposed `JSON.stringify` pattern for translation settings would double‑encode arrays/strings and break reads unless parsing logic is added. `packages/api/src/routes/config.ts:16-55`

**Overall assessment**
Not fully implementable as written. The core idea is workable, but it needs updates to align with the current pipeline (auto‑post flow, existing stages), API shape (`/articles` fields), config storage/parsing, and env schema before it will behave correctly.
